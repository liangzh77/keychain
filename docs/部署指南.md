# Keychain 轻量服务器部署指南

本文档按 `docs/go_web_项目改造指南.md` 的约定编写，用于把 Keychain 部署到由 Caddy 反向代理、systemd 管理的轻量服务器。

## 部署参数

| 项目 | 值 |
| --- | --- |
| 应用名 | `keychain` |
| 二进制名 | `keychain` |
| 构建入口 | `./cmd/server` |
| 服务端口 | `127.0.0.1:8081` |
| 健康检查 | `GET /healthz` |
| 服务器应用目录 | `/srv/apps/keychain` |
| 服务器裸仓库 | `/srv/git/keychain.git` |
| 服务器构建目录 | `/srv/build/keychain` |

## 线上目录

```text
/srv/apps/keychain/
  releases/
  current -> releases/当前版本
  previous -> releases/上一版本
  shared/
    app.db
    app.db-wal
    app.db-shm
    config.env
    data/
```

`systemd` 的 `WorkingDirectory` 必须是 `/srv/apps/keychain/shared`，这样默认的 `KEYCHAIN_DB_PATH=app.db` 会落到 shared 目录。

## config.env

在服务器写入 `/srv/apps/keychain/shared/config.env`，不要提交真实密钥：

```env
KEYCHAIN_ADMIN_USERNAME=admin
KEYCHAIN_ADMIN_PASSWORD=replace-with-real-password
KEYCHAIN_SESSION_SECRET=replace-with-long-random-secret
KEYCHAIN_RUNTIME_API_TOKEN=replace-with-long-random-runtime-token
KEYCHAIN_DB_PATH=app.db
KEYCHAIN_DATA_DIR=data
KEYCHAIN_ADDR=127.0.0.1:8081
```

## systemd

可参考仓库中的 `deploy/keychain.service`：

```bash
sudo cp deploy/keychain.service /etc/systemd/system/keychain.service
sudo systemctl daemon-reload
sudo systemctl enable keychain
sudo systemctl start keychain
```

## Caddy

线上域名：

```caddyfile
keychain.liangz77.cn {
    reverse_proxy 127.0.0.1:8081
}
```

## Git push 部署

服务器初始化：

```bash
sudo mkdir -p /srv/git /srv/build /srv/apps/keychain/shared/data
sudo chown -R deploy:deploy /srv/git /srv/build /srv/apps/keychain
sudo -u deploy git init --bare /srv/git/keychain.git
```

安装 hook：

```bash
sudo cp deploy/post-receive /srv/git/keychain.git/hooks/post-receive
sudo chmod +x /srv/git/keychain.git/hooks/post-receive
sudo chown deploy:deploy /srv/git/keychain.git/hooks/post-receive
```

允许 `deploy` 重启服务：

```text
deploy ALL=NOPASSWD: /usr/bin/systemctl restart keychain, /usr/bin/systemctl status keychain
```

本地添加生产 remote 后推送：

```bash
git remote add prod deploy@<SERVER_IP>:/srv/git/keychain.git
git push prod main
```

hook 会构建 `./cmd/server`，发布到 `/srv/apps/keychain/releases/<timestamp>/keychain`，切换 `current`，重启 `keychain`，并检查：

```bash
curl -fsS http://127.0.0.1:8081/healthz
```

健康检查失败时会切回 `previous` 并重启服务。

## 本地开发

```bash
cp .env.example .env
go test ./...
go run ./cmd/server
```

本地访问：

```text
http://127.0.0.1:8080
http://127.0.0.1:8080/healthz
```

`app.db`、`app.db-wal`、`app.db-shm`、`data/`、`config.env` 不应提交到 Git。
