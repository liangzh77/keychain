# 数据库设计

## 约定

- 数据库使用 SQLite。
- 启动时开启 WAL：`PRAGMA journal_mode = WAL`。
- 设置 busy timeout，降低并发写入时的锁冲突。
- 所有写入使用事务。
- 所有 SQL 使用参数化查询。
- 时间字段建议使用 UTC ISO8601 字符串或 Unix 毫秒时间戳，项目内保持一致。

## 表结构草案

### providers

```sql
CREATE TABLE providers (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  code TEXT NOT NULL UNIQUE,
  is_enabled INTEGER NOT NULL DEFAULT 1,
  rotation_strategy TEXT NOT NULL DEFAULT 'ROUND_ROBIN',
  round_robin_cursor INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

`rotation_strategy` 可选：

- `ROUND_ROBIN`
- `STICKY_FIRST_AVAILABLE`

### models

```sql
CREATE TABLE models (
  id TEXT PRIMARY KEY,
  provider_id TEXT NOT NULL,
  name TEXT NOT NULL,
  code TEXT NOT NULL,
  is_enabled INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (provider_id) REFERENCES providers(id) ON DELETE CASCADE,
  UNIQUE (provider_id, code)
);
```

### api_keys

```sql
CREATE TABLE api_keys (
  id TEXT PRIMARY KEY,
  provider_id TEXT NOT NULL,
  alias TEXT NOT NULL,
  secret_value TEXT NOT NULL,
  is_enabled INTEGER NOT NULL DEFAULT 1,
  is_available INTEGER NOT NULL DEFAULT 1,
  sort_order INTEGER NOT NULL DEFAULT 0,
  failure_count INTEGER NOT NULL DEFAULT 0,
  last_failed_at TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (provider_id) REFERENCES providers(id) ON DELETE CASCADE,
  UNIQUE (provider_id, alias)
);
```

### channels

```sql
CREATE TABLE channels (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  code TEXT NOT NULL UNIQUE,
  default_permission_mode TEXT NOT NULL DEFAULT 'DENY',
  user_management_mode TEXT NOT NULL DEFAULT 'EXTERNAL_MANAGED',
  is_enabled INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

`default_permission_mode` 可选：

- `ALLOW`
- `DENY`

`user_management_mode` 可选：

- `EXTERNAL_MANAGED`：外部系统自有用户系统，通过 Runtime API 同步用户。
- `KEYCHAIN_HOSTED`：用户系统托管在 Keychain，由 Runtime API 提供注册、登录、重置密码和注销能力。

### users

```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  channel_id TEXT NOT NULL,
  external_user_id TEXT NOT NULL,
  display_name TEXT NOT NULL,
  is_enabled INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (channel_id) REFERENCES channels(id) ON DELETE CASCADE,
  UNIQUE (channel_id, external_user_id)
);
```

### hosted_user_credentials

```sql
CREATE TABLE hosted_user_credentials (
  user_id TEXT PRIMARY KEY,
  password_hash TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

该表只保存 `KEYCHAIN_HOSTED` 渠道用户的密码哈希，不保存明文密码。托管用户的登录用户名存储在 `users.external_user_id` 中。

### channel_permission_defaults

```sql
CREATE TABLE channel_permission_defaults (
  id TEXT PRIMARY KEY,
  channel_id TEXT NOT NULL,
  provider_id TEXT NOT NULL,
  model_id TEXT NOT NULL,
  default_allowed INTEGER NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (channel_id) REFERENCES channels(id) ON DELETE CASCADE,
  FOREIGN KEY (provider_id) REFERENCES providers(id) ON DELETE CASCADE,
  FOREIGN KEY (model_id) REFERENCES models(id) ON DELETE CASCADE,
  UNIQUE (channel_id, provider_id, model_id)
);
```

### user_permissions

```sql
CREATE TABLE user_permissions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  provider_id TEXT NOT NULL,
  model_id TEXT NOT NULL,
  allowed INTEGER NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (provider_id) REFERENCES providers(id) ON DELETE CASCADE,
  FOREIGN KEY (model_id) REFERENCES models(id) ON DELETE CASCADE,
  UNIQUE (user_id, provider_id, model_id)
);
```

### user_key_permissions

```sql
CREATE TABLE user_key_permissions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  provider_id TEXT NOT NULL,
  key_id TEXT NOT NULL,
  allowed INTEGER NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (provider_id) REFERENCES providers(id) ON DELETE CASCADE,
  FOREIGN KEY (key_id) REFERENCES api_keys(id) ON DELETE CASCADE,
  UNIQUE (user_id, provider_id, key_id)
);
```

用户 key 权限用于限制某个用户在某个 provider 下可被分发的 key。未设置显式 key 权限时默认允许该 provider 下所有可用 key；一旦设置，运行时分发只会从 `allowed = 1` 的 key 中选择。

### dispatch_logs

```sql
CREATE TABLE dispatch_logs (
  id TEXT PRIMARY KEY,
  channel_id TEXT NOT NULL,
  user_id TEXT NOT NULL,
  provider_id TEXT NOT NULL,
  model_id TEXT NOT NULL,
  key_id TEXT NOT NULL,
  key_alias_snapshot TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'DISPATCHED',
  created_at TEXT NOT NULL,
  FOREIGN KEY (channel_id) REFERENCES channels(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (provider_id) REFERENCES providers(id),
  FOREIGN KEY (model_id) REFERENCES models(id),
  FOREIGN KEY (key_id) REFERENCES api_keys(id)
);
```

`key_alias_snapshot` 用于保留历史展示名称，即使后续 key 别名被修改，历史记录也能保持当时语义。

`status` 可选：

- `DISPATCHED`
- `FAILED_REPORTED`

### failure_reports

```sql
CREATE TABLE failure_reports (
  id TEXT PRIMARY KEY,
  dispatch_log_id TEXT NOT NULL,
  channel_id TEXT NOT NULL,
  user_id TEXT NOT NULL,
  provider_id TEXT NOT NULL,
  model_id TEXT NOT NULL,
  key_id TEXT NOT NULL,
  key_alias_snapshot TEXT NOT NULL,
  error_code TEXT,
  error_message TEXT,
  created_at TEXT NOT NULL,
  FOREIGN KEY (dispatch_log_id) REFERENCES dispatch_logs(id),
  FOREIGN KEY (channel_id) REFERENCES channels(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (provider_id) REFERENCES providers(id),
  FOREIGN KEY (model_id) REFERENCES models(id),
  FOREIGN KEY (key_id) REFERENCES api_keys(id)
);
```

### admin_sessions

```sql
CREATE TABLE admin_sessions (
  id TEXT PRIMARY KEY,
  session_token_hash TEXT NOT NULL UNIQUE,
  expires_at TEXT NOT NULL,
  created_at TEXT NOT NULL
);
```

Session token 原文只放在 httpOnly cookie 中，数据库保存 hash。

## 权限判断

输入：

- channel code 或 channel id。
- external user id 或 user id。
- provider code 或 provider id。
- model code 或 model id。

判断顺序：

1. channel 存在且启用。
2. user 存在且启用。
3. provider 存在且启用。
4. model 存在、属于 provider 且启用。
5. 查询 user_permissions，如果存在，返回其 allowed。
6. 查询 channel_permission_defaults，如果存在，返回其 default_allowed。
7. 使用 channels.default_permission_mode。
8. 如果仍无法判断，默认拒绝。

以上判断决定用户是否可以调用某个 provider/model。key 级授权在 key 分发阶段执行，只影响具体分发哪一个 key。

## Key 分发事务

分发 key 必须在事务内完成。

流程：

1. 解析 channel、user、provider、model。
2. 执行权限判断。
3. 根据 provider.rotation_strategy 选择可用 key。
4. 插入 dispatch_logs。
5. 对 `ROUND_ROBIN` 更新 provider.round_robin_cursor。
6. 提交事务。
7. 返回 key 明文和 dispatchLogId。

可用 key 条件：

```text
api_keys.provider_id = provider.id
AND api_keys.is_enabled = 1
AND api_keys.is_available = 1
AND COALESCE(user_key_permissions.allowed, 1) = 1
```

其中 `user_key_permissions` 按 `user_id + provider_id + key_id` 左连接；没有显式记录时默认允许。

排序：

```text
sort_order ASC, created_at ASC
```

### ROUND_ROBIN

读取 provider 的 `round_robin_cursor`，在可用 key 列表中选择对应位置的 key。成功分发后将 cursor 加一并对可用 key 数量取模。

如果并发较高，需要确保读取 cursor、选择 key、更新 cursor 在同一写事务内完成。

### STICKY_FIRST_AVAILABLE

选择排序最靠前的可用 key，不更新 cursor。

## 失败上报事务

失败上报也必须在事务内完成。

流程：

1. 根据 dispatchLogId 查询 dispatch_logs。
2. 插入 failure_reports。
3. 更新 dispatch_logs.status 为 `FAILED_REPORTED`。
4. 增加 api_keys.failure_count，更新 last_failed_at。
5. 根据后续实现策略决定是否将 api_keys.is_available 设为 0。MVP 可以先只记录失败次数，由管理员手动停用 key。
6. 提交事务。

## 展示规则

后台 key 列表：

- 显示 key 别名。
- 显示 masked key。
- 不默认显示完整明文 key。

历史记录：

- 显示 key_alias_snapshot。
- 不显示完整明文 key。

掩码建议：

```text
长度小于等于 8：全部显示为 ****
长度大于 8：前 3 位 + **** + 后 4 位
```
